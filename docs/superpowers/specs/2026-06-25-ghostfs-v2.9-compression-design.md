# ghostfs v2.9 — Kompresja (chunk extents + zlib): design

**Data:** 2026-06-25
**Część:** v2 (CoW B-tree). Trzeci cel (snapshoty✓, samonaprawa✓, kompresja).

## Decyzja: dwie ścieżki danych wg flagi kontenera (niskie ryzyko)
Kompresja pojedynczego bloku 4 KB nie oszczędza (min. 1 blok). Realna kompresja = **ekstenty
chunk** (N bloków logicznych → M<N bloków fizycznych). Aby NIE ruszać sprawdzonej ścieżki v2.4:
- **Kontener BEZ kompresji (domyślnie):** ekstenty per-blok `GH2_EXTENT_DATA` (v2.4, NIETKNIĘTE)
  → istniejące testy/zachowanie bez zmian.
- **Kontener `--compress` (flaga `GH2_SB_COMPRESS`):** ekstenty chunk `GH2_EXTENT_COMP`.
Brak mieszania w jednym kontenerze (`fs->compress` decyduje ścieżkę read/write/truncate).

## Format chunk extentu (`GH2_EXTENT_COMP`=6, wartość zmienna)
Chunk = `GH2_COMP_CHUNK` bloków (=8 → 32 KB). Klucz `(ino, GH2_EXTENT_COMP, chunk_aligned_off)`.
Wartość (zmienna długość, lista bloków — nieciągłe OK, brak kruchej alokacji ciągłej):
```c
struct gh2_cext_hdr { uint8_t comp_algo; uint8_t flags; uint16_t nblocks;
                      uint32_t raw_len; uint32_t comp_len; uint32_t csum; };
/* + uint64_t blocks[nblocks] */
```
- `comp_algo`: 0=raw, 1=zlib. `raw_len`: nieskompresowane bajty w chunku (≤32768; ostatni częściowy).
- `comp_len`: bajty zapisane (==raw_len gdy raw; <raw_len gdy skompresowane). `nblocks`=ceil(comp_len/4096).
- `blocks[]`: bloki fizyczne z (s)kompresowanymi danymi, po kolei. `csum`=crc32 zapisanych danych.
- Dziura = brak itemu → zera. Wartość ≤ 16 + 8×8 = 80 B (mieści się w liściu B-drzewa).

## Read (chunk)
offset X → chunk = align(X, CHUNK); lookup `(ino,GH2_EXTENT_COMP,chunk_off)`; brak → zera;
jest → czytaj `nblocks` bloków (gh_disk_read, deszyfr.), sklej `comp_len` B, weryfikuj `csum`
(niezgodność → -EIO; data DUP poza zakresem v2.9), jeśli zlib → uncompress do `raw_len`; zwróć
podzakres [X - chunk_off ...].

## Write (chunk, CoW + RMW)
- RMW chunku: wczytaj istniejący chunk (decompress) LUB zera (dziura); nałóż zapisywane bajty →
  bufor surowy `raw_len`.
- Spróbuj zlib `compress2`; jeśli `comp_len` mieści się w mniejszej liczbie bloków niż raw
  (oszczędza ≥1 blok) → zapisz skompresowane; inaczej raw (comp_algo=0).
- Alokuj `nblocks` (pojedynczo, nieciągłe), zapisz dane (gh_disk_write, szyfrowanie), zbuduj item.
- CoW: stare bloki chunku → defer_dec (free przy refcount 0); insert/update item. Atomowość per-op.
- inode.size = max; mtime.

## truncate / unlink / mark-sweep / snapshot
- truncate skróć: usuń chunki o off≥align(size) (free ich bloki defer_dec); częściowy ostatni
  chunk → RMW (obetnij raw_len, wyzeruj ogon, przepisz). rozszerz: tylko size.
- unlink/free: free WSZYSTKIE bloki wszystkich chunków (`gh2_inode_free_extents` rozszerzony).
- **mark-sweep/refcount** (`mark_extent_cb`): `GH2_EXTENT_DATA` → per-blok (jak dotąd);
  `GH2_EXTENT_COMP` → ref_inc KAŻDY blok z `blocks[]`. (Inaczej alloc nadpisze żywe dane chunku.)
- snapshot inc / delete dec / CoW: obejmują wszystkie bloki chunków (przez walk extents).

## Format/CLI/fasada
`gh2_fs_format` flaga COMPRESS → `GH2_SB_COMPRESS`; mount → `fs->compress`. CLI `format2
<k> <bloki> <inody> [--compress]` (kombinowalne z --dup); fasada gfs. zlib: `-lz` w Makefile LDLIBS.

## Testy (`tests/test_v2comp.c`)
1. **Oszczędność (sedno):** zapisz dużo danych ŚCIŚLIWYCH (np. zera/powtarzalne, 1 MB) do kontenera
   --compress → liczba ZAJĘTYCH bloków danych << liczba bloków logicznych (realna kompresja);
   dane NIEściśliwe (losowe) → fallback raw (brak wzrostu/strat, comp_algo=0).
2. **Round-trip:** różne rozmiary/offsety/dziury, częściowe chunki, granice chunków, niewyrównane
   read/write przez granice chunków — bajt-exact.
3. **RMW:** nadpisanie części chunku (CoW, recompress) → reszta chunku zachowana; sub-blokowy zapis.
4. **truncate** skróć/rozszerz/zero (chunki zwolnione, ogon wyzerowany, sparse=zera); unlink → bloki zwolnione.
5. **mark-sweep/refcount z chunkami (BRAMKA):** zajęte == bloki chunków + węzły + SB; refcount==
   mark-sweep liczy wszystkie bloki blocks[]; wyciek=0; CoW zwalnia stare bloki chunku.
6. **Snapshot + kompresja:** snapshot dzieli bloki chunków (refcount); izolacja; delete zwalnia wyłączne.
7. **Persystencja + crash:** write+commit+remount→dane identyczne; crash-sweep z --compress → fsck==0.
8. **Regresja:** kontener BEZ --compress = ścieżka v2.4 nietknięta (wszystkie istniejące testy 0 failed).
9. duży plik 4 MB ściśliwy/mieszany; ASan/UBSan czyste.

## Świadome ograniczenia
- Kompresja per-kontener (flaga), nie per-plik/atrybut. CHUNK=8 stały (RMW całego chunku przy
  sub-zapisie — koszt kompresji, jak btrfs). Data DUP/csum-heal danych poza zakresem (tylko detekcja
  -EIO). zlib (lz4/zstd = przyszłość przez comp_algo).
