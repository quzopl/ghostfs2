# ghostfs v2.2 — Alokator (mapa wolnej przestrzeni + CoW deferred-free): design

**Data:** 2026-06-25
**Część:** v2 (CoW B-tree). Master: `...ghostfs-v2-architecture-design.md`.

## Korekta dekompozycji (świadoma, uzasadniona)
Master przewidywał „v2.2 = alokator + drzewo ekstentów (refcounty)". **Przenoszę refcounty +
persystentne drzewo ekstentów do v2.7 (snapshoty)** — dopóki nie ma snapshotów, każdy blok ma
refcount=1, a samonośne drzewo ekstentów (alokacja jego własnych węzłów zmienia je samo —
pętla commitu) to najtrudniejsza i najbardziej bug-podatna część CoW FS. Budowanie jej przed
istnieniem współdzielenia = przedwczesna złożoność. v2.2 dostarcza **kompletny, poprawny
alokator** oparty o mapę wolnej przestrzeni odbudowywaną przy mount (mark-sweep). To NIE skrót
— to poprawny model (część realnych FS odbudowuje free-space przy mount); refcounty dojdą w
v2.7, gdzie free-on-CoW zmieni się na dec-refcount.

## Model
W świecie bez snapshotów (v2.2–v2.6): blok jest **zajęty wtedy i tylko wtedy, gdy osiągalny**
z zatwierdzonego superbloku (węzeł drzewa lub blok danych). Stan alokacji jest więc **niejawny
w strukturze drzew** — nie trzeba persystentnej mapy. Przy mount budujemy **mapę w pamięci**
(bitmapa, 1 bit/blok) przez mark-sweep wszystkich drzew. Alokacja/zwolnienie operują na
bitmapie w pamięci; trwałość wynika z commitu drzew (v2.5) + odbudowy mapy przy następnym mount.

## CoW deferred-free (kluczowe dla atomowości)
Przy mutacji CoW: nowe bloki alokowane NATYCHMIAST (oznacz zajęte w pamięci, by kolejne
alokacje w tej transakcji ich nie wzięły). Stare bloki ścieżki CoW **NIE są od razu
zwalniane** — żywe (zatwierdzone) drzewo wciąż na nie wskazuje aż do następnego commitu.
Trafiają na listę „do zwolnienia po commicie". Przy:
- **commit OK** → oznacz odroczone jako wolne (teraz nieosiągalne).
- **abort/rollback** → zwolnij bloki zaalokowane w tej transakcji (przywróć mapę), porzuć
  listę odroczonych.

## API (`src/v2/gh2_space.{h,c}`)
```c
struct gh2_space { uint8_t *bits; uint64_t nblocks; uint64_t hint; uint64_t nfree; };
int  gh2_space_init(struct gh2_space*, uint64_t nblocks);   /* wszystko wolne; bloki 0,1 (SB) zajęte */
void gh2_space_destroy(struct gh2_space*);
int  gh2_space_is_used(const struct gh2_space*, uint64_t blk);
void gh2_space_set(struct gh2_space*, uint64_t blk, int used);
int  gh2_space_alloc_one(struct gh2_space*, uint64_t *out);  /* znajdź wolny (hint), oznacz zajęty; -ENOSPC */

/* alokator transakcyjny (implementuje gh2_alloc z v2.1) z odroczonym free + rollback */
struct gh2_txn_alloc {
    struct gh2_space *space;
    uint64_t *defer_free; uint32_t ndf, dfcap;   /* stare bloki CoW: zwolnić po commicie */
    uint64_t *txn_alloced; uint32_t nta, tacap;   /* zaalokowane w txn: zwolnić przy abort */
    int oom;
};
int  gh2_txn_alloc_init(struct gh2_txn_alloc*, struct gh2_space*);
void gh2_txn_alloc_destroy(struct gh2_txn_alloc*);
struct gh2_alloc gh2_txn_alloc_vtable(struct gh2_txn_alloc*);  /* {alloc,free,ctx} dla B-drzewa */
int  gh2_txn_alloc_commit(struct gh2_txn_alloc*);   /* zwolnij odroczone; wyczyść listy */
void gh2_txn_alloc_abort(struct gh2_txn_alloc*);    /* zwolnij txn_alloced; przywróć mapę */
```
`alloc(ctx,*out)` = `gh2_space_alloc_one` + zapis do `txn_alloced`. `free(ctx,blk)` = dopisz do
`defer_free` (NIE zwalnia od razu).

## Mark-sweep (budowa mapy z drzew)
```c
/* odwiedź KAŻDY blok-węzeł drzewa (internal+leaf) — do mark-sweep mapy */
int gh2_btree_walk_nodes(struct gh_dev*, const struct gh2_bptr *root,
                         int (*cb)(uint64_t block, void *ctx), void *ctx);
int gh2_space_build_from_tree(struct gh_dev*, struct gh2_space*, const struct gh2_bptr *root);
```
(W v2.3/v2.6: mark-sweep po wszystkich drzewach + blokach danych ekstentów (v2.4). v2.2 testuje
budowę z jednego B-drzewa.)

## Testy (`tests/test_v2space.c`)
1. `gh2_space`: init (bloki 0,1 zajęte, reszta wolna, nfree poprawne); set/is_used; alloc_one
   zwraca tylko wolne, hint postępuje; pełne → -ENOSPC; nfree spójne.
2. **deferred-free + abort:** alloc N (oznaczone zajęte natychmiast); free M (NIE zwolnione —
   wciąż zajęte); commit → teraz wolne. Inny scenariusz: alloc N, abort → wszystkie N znów
   wolne (mapa jak przed txn); odroczone porzucone.
3. **mark-sweep:** zbuduj B-drzewo (gh2_btree) realnym alokatorem (txn alloc + commit); walk_nodes
   liczy bloki; build_from_tree do świeżej mapy → DOKŁADNIE te same bloki zajęte co użyte przez
   drzewo (porównaj z mapą po budowie). Brak rozbieżności (fundament odbudowy przy mount).
4. **integracja B-drzewo + alokator:** insert/delete dużo z txn alloc; po każdym commit mapa
   spójna; brak double-alloc (żaden żywy węzeł nie współdzieli bloku); brak wycieku (po
   usunięciu wszystkiego i commitach, zajęte == żywe węzły).
5. ASan/UBSan czyste; brak wycieków pamięci.

## Świadome decyzje / ograniczenia
- Brak refcountów/drzewa ekstentów → v2.7 (snapshoty). free-on-CoW bezpieczne (refcount≡1).
- Mapa w pamięci: 1 bit/blok (~32 MB/TB) — akceptowalne; odbudowa O(metadanych) przy mount.
- hint-based first-fit (jak v1) — wystarcza; lepsza polityka to przyszłość.
