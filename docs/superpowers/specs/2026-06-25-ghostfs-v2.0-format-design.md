# ghostfs v2.0 — Format & superblok: design

**Data:** 2026-06-24
**Część:** v2 (CoW B-tree) — fundament. Master: `...ghostfs-v2-architecture-design.md`.

## Cel
Format `GHOSTFS\x02` + podwójny superblok (ping-pong) z atomowym commitem przez podmianę
slotu. To fundament atomowości v2 (zastępuje dziennik). Bez drzew jeszcze (stub: bump
alokator + zerowy wskaźnik korzenia).

## Struktury (`src/v2/gh2_format.h`)
- `GH2_MAGIC "GHOSTFS\x02"` (8B), `GH2_BLOCK_SIZE 4096`.
- `struct gh2_bptr { u64 block; u64 dup_block; u32 csum; u32 _pad; u64 generation; }`
  — wskaźnik bloku z duplikatem (samonaprawa) i sumą (integralność). W v2.0 zerowy (stub).
- `struct gh2_superblock` (mieści się w 1 bloku):
  `char magic[8]; u64 generation; u64 total_blocks; u32 block_size; u32 flags;
   gh2_bptr root_tree; u64 next_free; u64 root_dir_ino; u8 uuid[16];
   u8 enc_salt[16]; u8 enc_verifier[32]; u64 reserved[16]; u32 sb_csum;`
  (`sb_csum` ostatnie pole; suma liczona ze wszystkiego przed nim).

## Sloty i ping-pong
Dwa sloty SB: blok **0** i blok **1**. Slot dla generacji `g` = `g & 1`. Commit(g) pisze
TYLKO slot `g&1` → nigdy nie nadpisuje slotu z `g-1` (przeciwna parzystość). Mount: czytaj
oba, waliduj magic+csum, wybierz najwyższą `generation`. Region danych zaczyna się od bloku
**2** (`next_free` startuje od 2).

## API (`src/v2/gh2_super.{h,c}`)
- `uint32_t gh2_sb_csum(const struct gh2_superblock*)` — crc32 (csum.c) z bajtów przed `sb_csum`.
- `int gh2_format(struct gh_dev*, uint64_t total_blocks, uint32_t flags)` — zapisz SB gen=1
  do OBU slotów (bootstrap), `next_free=2`, `root_tree` zerowy. Raw `pwrite` (SB nieszyfrowany).
- `int gh2_mount(struct gh_dev*, struct gh2_superblock *out)` — czytaj oba sloty, wybierz
  najwyższą ważną generację (magic+csum OK); brak ważnego → `-EINVAL`.
- `int gh2_commit_super(struct gh_dev*, struct gh2_superblock *sb)` — `sb->generation++`,
  przelicz csum, raw `pwrite` do slotu `gen&1`, `fsync`. (W v2.5 to ostatni krok commitu drzewa.)

Reużycie: `gh_dev_create/open/close` (v1 block.c), `gh_crc32` (csum.c). SB przez raw
`pread/pwrite(dev->fd, ..., slot*4096)` — bez szyfrowania (SB trzyma sól).

## Testy (`tests/test_v2super.c`) — bramka crash-safety
1. format → mount: generation==1, magic OK, next_free==2, total_blocks zgodne.
2. commit ×N: generation rośnie; po remount widać najnowszą; sloty ping-pong (gen parzysta→0,
   nieparzysta→1).
3. **CRASH (bramka):** commit do gen N (slot N&1), nadpisz TEN slot śmieciami → mount wybiera
   gen N-1 z drugiego slotu (ważny, spójny). Nadpisz drugi slot zamiast → mount wybiera N.
4. csum: przekręć bit w slocie → ten slot odrzucony, wybrany drugi. Oba zepsute → -EINVAL.
5. ASan/UBSan czyste.

## Build
`src/v2/*.c` dodane do `CORE` w Makefile; `-Isrc` w CFLAGS (include `"v2/gh2_format.h"`,
`"block.h"`). Bez zmian symboli v1 (prefix `gh2_`).
