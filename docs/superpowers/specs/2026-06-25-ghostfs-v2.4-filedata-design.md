# ghostfs v2.4 — Dane plików (ekstenty): design

**Data:** 2026-06-25
**Część:** v2 (CoW B-tree). Buduje read/write/truncate plików na drzewie FS (v2.3).

## Model: ekstent per blok 4 KB
Treść pliku = itemy **EXTENT_DATA** w drzewie FS, JEDEN na blok 4 KB pliku (proste, wspiera
sparse + CoW + losowe zapisy; multi-block extenty = przyszła optymalizacja):
- Klucz: `(ino, GH2_EXTENT_DATA=3, file_off)` gdzie `file_off` = offset bloku w pliku
  (wielokrotność 4096).
- Wartość:
  ```c
  struct gh2_extent { uint64_t disk_block; uint64_t dup_block; uint32_t csum;
      uint16_t comp_algo; uint16_t flags; uint32_t raw_len; uint32_t comp_len; };
  ```
  `disk_block` = blok fizyczny z danymi; `csum` = crc32 plaintextu bloku; `raw_len` = ważne
  bajty (ostatni blok < 4096); `dup_block`=0 (v2.8); `comp_algo`=0/`comp_len`=raw_len (v2.9).
  **Dziura** = brak itemu → odczyt zwraca zera.

## I/O danych
- Zapis bloku: alokuj nowy blok danych (CoW — jeśli ekstent istnieje, stary disk_block →
  defer_free), `gh_disk_write` (szyfrowanie), `csum=gh_crc32(plaintext)`, insert/update
  EXTENT_DATA. Częściowy blok → read-modify-write (odczyt starego/zer + nałożenie). Wszystko w
  ramach atomowości per-operacja (savepoint/rollback z v2.3) na lokalnym root.
- Odczyt bloku: lookup EXTENT_DATA(ino,3,off); jest → `gh_disk_read(disk_block)`, weryfikuj
  csum (niezgodność → dup_block jeśli !=0, inaczej -EIO), zwróć raw_len bajtów; brak → zera.

## Operacje (`gh2_fs_*`)
- `gh2_fs_write(fs, path, buf, len, off)` → ssize_t — pętla po blokach; aktualizuj inode.size
  (max(size, off+len)) i mtime; tylko GH2_FT_FILE (inne → -EISDIR/-EINVAL). Atomowa per-wywołanie.
- `gh2_fs_read(fs, path, buf, len, off)` → ssize_t — ogranicz do inode.size; dziury = zera.
- `gh2_fs_truncate(fs, path, size)` — rozszerz: tylko inode.size (sparse). Skróć: usuń
  EXTENT_DATA o off ≥ ceil(size) (free disk_block defer), popraw ostatni częściowy blok
  (raw_len / wyzeruj ogon), inode.size=size. (Zastępuje truncate-tylko-size z v2.3.)
- `gh2_fs_unlink`/usunięcie i-węzła pliku: usuń WSZYSTKIE EXTENT_DATA (free disk_block defer) +
  INODE_ITEM. (Rozszerza v2.3 unlink, który nie znał ekstentów.)
- `gh2_inode_free_extents(fs, &root, ino)` — helper: range-scan EXTENT_DATA(ino,3,*), free
  każdy disk_block (defer), usuń itemy.

## Mark-sweep rozszerzenie
Mapa wolnej przestrzeni MUSI teraz oznaczać też **bloki danych** (disk_block ekstentów), nie
tylko węzły drzewa. `gh2_space_build_from_tree` / mark-sweep: dla każdego leaf-itemu typu
EXTENT_DATA oznacz `disk_block` (i dup_block) zajęty. (Walk leaf-itemów, nie tylko węzłów.)
→ rozszerz `gh2_btree_walk_nodes` lub dodaj walk itemów; mark-sweep oznacza węzły + bloki danych.

## Testy (`tests/test_v2fs.c` rozszerzenie)
1. write/read round-trip: różne rozmiary (sub-blok, wielo-blok, niewyrównane offsety, częściowy
   ostatni blok); read poza size = 0/obcięte; dziury (zapis na offsecie) → zera między.
2. nadpisanie (CoW: stary blok zwolniony, nowy; brak wycieku); read-modify-write częściowego bloku.
3. truncate: skrócenie (ekstenty/bloki zwolnione, ogon wyzerowany), rozszerzenie (sparse, zera),
   do zera, round-trip.
4. unlink pliku z danymi → wszystkie bloki danych zwolnione (mapa==mark-sweep, wyciek=0).
5. **persystencja:** write+commit+remount → dane identyczne; mark-sweep oznacza bloki danych
   (mapa po remount == rzeczywista); wyciek bloków=0 (zajęte == węzły + bloki danych).
6. csum danych: przekręć bit w disk_block → read -EIO (dup=0).
7. **spójność fsck:** rozszerz walidator o bloki danych (każdy disk_block w mapie; brak
   współdzielenia bloku danych między i-węzłami; suma bajtów size spójna z ekstentami).
8. duży plik (np. 4 MB = 1024 bloków) write/read/truncate; ASan czyste; brak wycieków.

## Świadome ograniczenia
- 1 item/blok (bez multi-block extentów) — proste, poprawne; kompresja v2.9 rozszerzy wartość.
- dup/comp pola zarezerwowane (v2.8/v2.9).
