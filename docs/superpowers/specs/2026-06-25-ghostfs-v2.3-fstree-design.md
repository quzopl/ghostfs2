# ghostfs v2.3 — Drzewo FS + i-węzły: design

**Data:** 2026-06-25
**Część:** v2 (CoW B-tree). Master: `...ghostfs-v2-architecture-design.md`.
Buduje właściwą semantykę systemu plików (i-węzły, katalogi, operacje metadanych) na
fundamencie v2.0–v2.2.

## Stan montażu v2
```c
struct gh2_fs {
    struct gh_dev dev;
    struct gh2_superblock sb;
    struct gh2_bptr fs_root;        /* korzeń drzewa FS (sb.root_tree; drzewo korzeni → v2.7) */
    struct gh2_space space;         /* mapa wolnej przestrzeni (mark-sweep przy mount) */
    struct gh2_txn_alloc alloc;     /* alokator transakcyjny */
    uint64_t next_ino;              /* z sb (rezerwa) lub liczone */
};
```

## Itemy drzewa FS (klucz = objectid,type,offset)
- **INODE_ITEM** `(ino, GH2_INODE_ITEM=1, 0) → struct gh2_inode`:
  ```c
  struct gh2_inode { uint16_t type; uint16_t mode; uint32_t uid; uint32_t gid;
      uint32_t nlink; uint64_t size; uint64_t atime,mtime,ctime; uint64_t rdev;
      uint64_t generation; uint32_t flags; uint32_t _pad; };
  ```
  `type`: reuzycie semantyki v1 (FILE=1,DIR=2,SYMLINK=3,FIFO=4,SOCK=5,CHR=6,BLK=7) — własne
  stałe GH2_FT_*. `rdev` dla urządzeń. Korzeń = ino 1 (DIR), tworzony przy format.
- **DIR_ITEM** `(parent_ino, GH2_DIR_ITEM=2, name_hash) → spakowana lista wpisów`:
  ```c
  struct gh2_dirent { uint64_t ino; uint8_t ftype; uint8_t _pad; uint16_t name_len; /* name[] */ };
  ```
  `name_hash` = 64-bit FNV-1a nazwy. Kolizje hash → wiele wpisów spakowanych w jednej
  wartości (lookup skanuje po dokładną nazwę; add dokleja; remove wyjmuje, usuwa item gdy pusty).
- **SYMLINK_DATA** `(ino, GH2_SYMLINK_DATA=4, 0) → cel (bajty)` — dla symlinków (krótkie, ≤ max
  wartości B-drzewa). (Ekstenty danych plików → v2.4, type GH2_EXTENT_DATA=3 zarezerwowany.)

`next_ino`: w superbloku (`root_dir_ino` przemianować/dodać pole `next_ino`); start 2 (1=root).
Numery i-węzłów nie recyklowane (u64).

## Operacje (`gh2_fs_*`, lustro API v1 `gh_fs_*`, te same errno)
- `gh2_fs_format(dev, total_blocks, flags)` — gh2_format (v2.0) + utwórz pusty fs-tree +
  i-węzeł root (ino 1, DIR, nlink 2, mode 0755) + commit. `gh2_fs_mount`/`gh2_fs_unmount`.
- **lookup/path resolve:** `gh2_path_resolve(fs, path, *ino)` — od root, split na komponenty,
  każdy przez DIR_ITEM lookup w bieżącym katalogu; `.`/`..` obsłużone; -ENOENT/-ENOTDIR.
- `gh2_fs_getattr(fs, path, *inode, *ino)`.
- `gh2_fs_create(fs,path,mode)` / `gh2_fs_mkdir` / `gh2_fs_mknod(path,mode,rdev)` /
  `gh2_fs_symlink(target,path)` — alokuj ino, wstaw INODE_ITEM, wstaw DIR_ITEM w rodzicu,
  zwiększ nlink rodzica (mkdir: +1 za `..`; nowy DIR nlink=2). -EEXIST gdy istnieje.
- `gh2_fs_readdir(fs,path,cb)` — range-scan DIR_ITEM rodzica, emituj `.`,`..`,wpisy.
- `gh2_fs_unlink` — usuń DIR_ITEM; nlink-- i-węzła; przy 0 usuń INODE_ITEM (+SYMLINK_DATA).
  -EISDIR dla katalogu.
- `gh2_fs_rmdir` — tylko pusty (range-scan brak wpisów poza .. own); usuń DIR_ITEM + INODE_ITEM;
  nlink rodzica--. -ENOTEMPTY.
- `gh2_fs_link(old,new)` — hardlink: nowy DIR_ITEM → ten sam ino; nlink++. -EPERM dla katalogu.
- `gh2_fs_rename(old,new[,flags])` — usuń stary DIR_ITEM, wstaw nowy (ten sam ino); obsłuż
  nadpisanie celu (unlink celu); ochrona przed pętlą (katalog do własnego poddrzewa).
- `gh2_fs_readlink`, `gh2_fs_chmod/chown/utimens`, `gh2_fs_truncate` (na razie tylko size +
  ewentualne obcięcie — dane plików w v2.4), `gh2_fs_statfs` (z mapy: nfree).

## Commit (uproszczony; pełny w v2.5)
Każda operacja mutuje fs-tree (CoW → nowy fs_root). `gh2_fs_commit(fs)`:
1. `fsync(dev->fd)` — utrwal nowe bloki drzewa (gh_disk_write = write-through → w cache OS),
2. `sb.root_tree = fs_root` (+sb.generation prze gh2_commit_super), zapisz SB + fsync (atomowa
   podmiana korzenia — v2.0),
3. `gh2_txn_alloc_commit` — zwolnij stare bloki CoW (teraz nieosiągalne).
Awaria przed krokiem 2 → stary SB → stary fs_root → stary spójny stan (nowe bloki nieosiągalne,
przy remount mark-sweep je „odzyska" jako wolne). Operacje mogą commit-ować po każdej (test) lub
batch (v2.5). Mount: mark-sweep z sb.root_tree odbudowuje mapę.

## Rozszerzenie B-drzewa (potrzebne tu)
`gh2_btree_iterate_range(dev, root, min_key, max_key, cb, ctx)` — iteracja in-order od
pierwszego klucza ≥ min do ≤ max (readdir, sprawdzenie pustości katalogu). (Zejście do min +
in-order do max.)

## Testy (`tests/test_v2fs.c`)
1. format→mount: root istnieje (getattr "/" = DIR, nlink 2); statfs nfree sensowne.
2. create/mkdir/lookup/getattr; zagnieżdżone katalogi; -EEXIST; -ENOENT; -ENOTDIR.
3. readdir: `.`,`..`+wpisy; wiele wpisów (wymusza kolizje hash → pakowanie); kolejność.
4. unlink/rmdir: nlink, -EISDIR/-ENOTEMPTY; usunięcie i-węzła przy nlink 0; rmdir nlink rodzica.
5. hardlink: nlink++, oba wpisy → ten sam ino; unlink jednego → drugi żyje.
6. symlink/readlink; rename (w tym nadpisanie, między katalogami, ochrona pętli); chmod/chown/
   utimens; mknod (urządzenie: rdev round-trip); truncate (size).
7. **persystencja:** sekwencja operacji → `gh2_fs_commit` → unmount → remount → stan zachowany;
   mark-sweep mapa spójna; brak wycieku bloków (zajęte == żywe węzły fs-tree).
8. **spójność (fsck-poziom):** po losowej sekwencji operacji walidacja: każdy DIR_ITEM →
   istniejący INODE_ITEM; nlink i-węzła == liczba wpisów (+ katalog: 2 + podkatalogi); brak
   sierot; mapa == mark-sweep.
9. ASan/UBSan czyste; brak wycieków pamięci.

## Świadome ograniczenia v2.3
- Dane plików (read/write) → v2.4 (truncate tu tylko ustawia size).
- Snapshoty/refcounty → v2.7; commit uproszczony (pełny crash-sweep w v2.5).
- xattr → później (zarezerwować typ GH2_XATTR_ITEM).
- next_ino monotoniczny (bez recyklingu).
