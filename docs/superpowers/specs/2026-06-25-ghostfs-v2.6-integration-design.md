# ghostfs v2.6 — Integracja FUSE/CLI (montowalny v2): design

**Data:** 2026-06-25
**Część:** v2 (CoW B-tree). Zamienia rdzeń v2 (działający na API gh2_fs_*) w MONTOWALNY system
plików: FUSE i CLI wykrywają magic v1(`\x01`)/v2(`\x02`) i kierują do gh_fs_*/gh2_fs_*.

## Fasada `struct gfs` (`src/v2/gh2_vfs.{h,c}`)
Wspólny front nad v1 i v2:
```c
struct gfs_attr { uint16_t type, mode; uint32_t uid, gid, nlink;
                  uint64_t size, atime, mtime, ctime, rdev; };   /* type: FILE=1..BLK=7 (wspólne) */
struct gfs { int version; /*1 lub 2*/ struct gh_fs v1; struct gh_dev dev2; struct gh2_fs v2; };
```
- `int gfs_detect(const char *path)` — open, pread bloku 0, sprawdź magic; zwróć 1/2/-errno.
- `int gfs_mount(struct gfs*, const char *path, const char *key_or_NULL)` — detect; v1 →
  gh_fs_mount/gh_fs_mount_key; v2 → gh_dev_open(dev2)+gh2_fs_mount(&v2,&dev2) (klucz: ustaw
  szyfr. na dev2 jak v1 — reuzycie crypto). `void gfs_unmount(struct gfs*)`.
- Każda operacja `gfs_*` przełącza po `version`:
  getattr(→gfs_attr), create, mkdir, mknod, symlink, readlink, link, unlink, rmdir, rename,
  chmod, chown, utimens, truncate, read, write, readdir(cb), statfs, sync(=v1 gh_fs_sync / v2
  gh2_fs_commit), fsck. **xattr:** v1→gh_fs xattr; v2→`-ENOTSUP` (v2 bez xattr — przyszłość).
- Mapowanie błędów/typów wspólne (oba rdzenie używają tych samych wartości type/errno).

## FUSE (`src/fuse_main.c`)
- Globalny `g_fs` → `struct gfs`. Mount: `gfs_mount` (auto-detect; prompt hasła przy -EACCES jak
  dotąd). Handlery `gf_*` wołają `gfs_*` zamiast bezpośrednio `gh_fs_*`. getattr mapuje
  `gfs_attr`→`struct stat` (S_IFDIR/LNK/REG/FIFO/SOCK/CHR/BLK + rdev). flock + rwlock (D) bez
  zmian. `gf_flush` leniwy (M); `gf_fsync`/unmount → `gfs_sync` (v1 sync / v2 commit).
- Strojenie I/O (Q: max_write/splice/writeback) bez zmian. v2 dziedziczy.

## CLI (`src/cli.c`)
- Komendy używają `gfs_*` (auto-detect formatu kontenera). `cli_mount`→`gfs_mount` (prompt hasła).
- **format:** `format <plik> <bloki> <inody> [--v2]` LUB nowa komenda `format2` — tworzy v2
  (gh2_fs_format). Domyślnie v1 (kompatybilność). `df`/`fsck`/`stat` działają na v2 (statfs/fsck/
  getattr przez fasadę). put/get/ls/mkdir/rm/mv/ln/lns/chmod/mknod/truncate → fasada.

## Trwałość / commit
v2: ops mutują w pamięci; `gfs_sync` (→gh2_fs_commit) na fsync/unmount/zamknięcie CLI. FUSE
leniwy flush (M). CLI: każda komenda mutująca → operacja + `gfs_sync` przed unmount (jak v1
commituje przez unmount). Mount v2 odbudowuje mapę mark-sweep.

## Świadome ograniczenia
- v2 xattr → -ENOTSUP (przyszłość). FUSE setxattr/getxattr na v2 zwróci ENOTSUP (toleruje to
  większość narzędzi).
- format domyślnie v1 (kompatybilność wsteczna); v2 przez --v2/format2.
