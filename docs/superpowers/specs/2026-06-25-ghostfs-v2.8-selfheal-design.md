# ghostfs v2.8 — Samonaprawa (DUP metadane + read-repair): design

**Data:** 2026-06-25
**Część:** v2 (CoW B-tree). Drugi z trzech celów (snapshoty✓, samonaprawa, kompresja).

## Model
Metadane (węzły B-drzewa) zapisywane w DWÓCH kopiach (pole `dup_block` w `gh2_bptr` istnieje od
v2.1). Odczyt: weryfikuj sumę `block`; niezgodność → czytaj `dup_block`, weryfikuj; sukces →
zwróć + (best-effort) **napraw** zepsutą kopię (przepisz dobrą treść). Oba złe → -EIO. Bitrot
jednej kopii węzła jest niewidoczny dla użytkownika i naprawiany przy odczycie.

## Co JUŻ jest (od v2.1, projektowane pod to)
- `gh2_node_read` (gh2_btree.c:105): czyta `block`; przy złej sumie próbuje `dup_block` (jest!).
- `cow_free` (gh2_btree.c:251): zwalnia OBIE kopie (`block`+`dup_block`) — jest!
- Ekstenty danych: read-fallback dup + refcount inc/dec dup (gh2_fs.c) — jest.

## Co dożywa v2.8
1. **Flaga DUP**: `GH2_SB_DUP_META` (istnieje w gh2_format.h) ustawiana opcjonalnie przy format.
   Mount czyta → `dev`/`gh2_fs` wie. `gh2_alloc` dostaje pole `int dup_meta` (vtable z gh2_txn_alloc
   ustawia z flagi). **OFF domyślnie** (zero zmian w istniejących testach/block-math). `format2 --dup`/
   CLI włącza.
2. **`gh2_node_write`**: jeśli `a->dup_meta` → alokuj 2 bloki (przez vtable), zapisz węzeł do OBU
   (`gh_disk_write` ×2), ustaw `out_bptr->block` + `out_bptr->dup_block` + csum. Błąd 2. alokacji/
   zapisu → zwolnij oba + zwróć błąd. (DUP off → 1 blok, dup_block=0 jak dotąd.)
3. **`walk_node`** (mark-sweep): raportuj też `node->dup_block` (cb na obie kopie), by mark-sweep/
   refcount liczyły OBIE kopie każdego węzła. (Inaczej kopia dup byłaby „niezaznaczona" → alloc by
   ją nadpisał — korupcja.)
4. **read-repair** w `gh2_node_read`: gdy `block` zła suma a `dup_block` dobra → zwróć dobre dane
   ORAZ best-effort `gh_disk_write(block, dobre)` (napraw; ignoruj błąd zapisu — odczyt nadal OK).
   (Dla węzła współdzielonego snapshotem: przepisanie TEJ SAMEJ treści to naprawa bitrotu, nie
   narusza CoW — wszyscy współdzielący zyskują.)
5. Format/CLI/fasada: `gh2_fs_format` + opcja DUP; `format2 --dup`/`--selfheal`; gfs przekazuje.

## Refcounty/snapshoty z DUP
Węzeł = 2 bloki fizyczne (block+dup_block), oba referencyjne przez ten sam bptr. mark-sweep/
refcount liczą OBA (walk_node raportuje oba). Snapshot inc / delete dec / CoW free obejmują OBA
(cow_free już zwalnia oba; refmap przez walk_node liczy oba). Brak zmian w logice CoW B-drzewa
(tylko node_write/read/walk_node — I/O węzła, NIE algorytm split/merge).

## Testy (`tests/test_v2heal.c`)
1. Format z DUP → każdy węzeł ma `dup_block != 0` i obie kopie identyczne (bajt-exact).
2. **Read-repair:** przekręć bit w `block` węzła (surowo) → odczyt drzewa zwraca POPRAWNE dane
   (z dup) BEZ -EIO; po odczycie `block` NAPRAWIONY (suma znów dobra). Przekręć `dup_block` →
   odczyt z block OK. Przekręć OBIE → -EIO.
3. **Mark-sweep/refcount z DUP:** zajęte == 2× węzły + bloki danych + SB; refcount==mark-sweep
   liczy obie kopie; wyciek=0; CoW zwalnia obie stare kopie.
4. **Snapshot z DUP:** snapshot dzieli obie kopie każdego węzła (refcount==2 dla block i dup_block);
   delete zwalnia obie wyłączne; izolacja zachowana.
5. **Realistyczna samonaprawa:** zbuduj FS z plikami/katalogami (DUP), przekręć losowe bity w
   losowych blokach-węzłach (jedna kopia każdego) → cały odczyt FS poprawny (samonaprawa), fsck==0.
6. **Regresja:** DUP OFF (domyślne formaty) → zachowanie identyczne jak v2.0-2.7 (dup_block=0,
   istniejące testy 0 failed); B-drzewo logika CoW NIETKNIĘTA.
7. Crash + DUP: crash-sweep z DUP → fsck==0, obie kopie spójne.
ASan/UBSan czyste.

## Świadome ograniczenia
- DUP tylko metadane (węzły); dane single (DUP danych = 2× miejsca; opcja/przyszłość).
- DUP off domyślnie (uniknięcie zmian block-math w testach); default-on = trywialny follow-up.
- read-repair best-effort (jeśli zapis naprawy zawiedzie, odczyt nadal zwraca dobre dane).
