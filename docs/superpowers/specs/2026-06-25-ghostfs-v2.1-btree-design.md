# ghostfs v2.1 — CoW B-tree (prymityw): design

**Data:** 2026-06-24
**Część:** v2 (CoW B-tree) — fundament indeksowania. Master: `...ghostfs-v2-architecture-design.md`.
Na tym drzewie stoją WSZYSTKIE drzewa v2 (FS, ekstenty, sumy, korzenie) → musi być wzorcowo
poprawne. Bramka: testy property-based vs model referencyjny + recenzja adwersaryjna.

## Model
Generyczne **copy-on-write** B-drzewo mapujące **klucz → wartość (bajty, zmienna długość)**.
Każda mutacja (insert/update/delete) **nie nadpisuje** węzłów — alokuje nowe bloki dla
zmienionej ścieżki w górę do korzenia i **zwraca nowy wskaźnik korzenia** (`gh2_bptr`). Stary
korzeń pozostaje ważny (fundament snapshotów). Węzły z sumą (w `bptr`) + opcjonalnym
duplikatem (samonaprawa v2.8).

## Klucz
```c
struct gh2_key { uint64_t objectid; uint8_t type; uint64_t offset; };
```
Porządek: leksykograficznie (objectid, potem type, potem offset). Klucze **unikalne** w
drzewie (insert istniejącego = update wartości). `gh2_key_cmp(a,b) -> -1/0/1`.

## Układ węzła (blok 4096 B) — determinizm pod sumę
Bufor węzła **memset 0** przed wypełnieniem; pola pisane jawnie (padding zawsze 0 →
powtarzalna suma). Nagłówek (stały, na offsecie 0):
```c
struct gh2_node_hdr {
    uint8_t  level;        /* 0 = liść, >0 = wewnętrzny */
    uint8_t  _pad[3];
    uint32_t nritems;
    uint64_t generation;
    uint64_t owner;        /* objectid drzewa (diagnostyka) */
};
```
- **Liść (level 0):** za nagłówkiem tablica nagłówków itemów rosnąca DO PRZODU; dane itemów
  rosnące OD KOŃCA bloku do tyłu (układ btrfs):
  ```c
  struct gh2_leaf_item { struct gh2_key key; uint32_t data_off; uint32_t data_len; };
  ```
  `data_off` = offset danych od początku bloku. Wolne miejsce = między końcem tablicy itemów
  a początkiem danych. Pojemność = zmienna (zależna od sumy długości wartości).
- **Wewnętrzny (level>0):** za nagłówkiem tablica:
  ```c
  struct gh2_internal_ptr { struct gh2_key key; struct gh2_bptr child; };
  ```
  `key` = NAJMNIEJSZY klucz w poddrzewie `child`. Lookup: największe `i` z `key[i] <= szukany`
  → zejdź do `child[i]`. Pojemność stała = `(4096 - hdr) / sizeof(gh2_internal_ptr)`.

Stałe: `GH2_LEAF_MIN`/`GH2_INT_MIN` (połowa pojemności — próg merge/borrow przy delete).
Wartość większa niż mieści się w pustym liściu → `-EFBIG` (duże wartości to zadanie ekstentów
v2.4, nie B-drzewa; tu wartości są małe: i-węzły/wpisy/itemy ekstentów ≤ ~kilkaset B).

## I/O węzłów (reuzycie warstwy szyfrowania v1, BEZ dziennika v1)
- **Zapis:** `gh2_node_write(dev, alloc, buf, gen, &out_bptr)` — alokuj blok (alloc), policz
  `csum = gh_crc32(buf, 4096)` (plaintext), `gh_disk_write(dev, blk, buf)` (szyfrowanie+cache,
  bez txn/csum v1), ustaw `out_bptr = {block, dup=0, csum, generation=gen}`. (DUP w v2.8.)
- **Odczyt:** `gh2_node_read(dev, &bptr, buf)` — `gh_disk_read(dev, bptr.block, buf)`
  (deszyfr.), sprawdź `gh_crc32(buf)==bptr.csum`; niezgodność → spróbuj `bptr.dup_block`
  (jeśli !=0); oba złe → `-EIO`.

## Alokator (pluggable; prawdziwy w v2.2)
```c
struct gh2_alloc {
    int  (*alloc)(void *ctx, uint64_t *out_block);   /* nowy wolny blok */
    void (*free)(void *ctx, uint64_t block);          /* zwolnij (CoW: stary węzeł) */
    void *ctx;
};
```
v2.1 dostarcza **stub**: `gh2_bump_alloc` (licznik next_free + prosta lista wolnych z free()),
do testów. v2.2 zastąpi alokatorem ekstentowym z refcountami.

## API B-drzewa
```c
/* pusty korzeń: nowy liść 0 itemów */
int gh2_btree_create(struct gh_dev*, struct gh2_alloc*, uint64_t owner, uint64_t gen,
                     struct gh2_bptr *out_root);
/* lookup: kopiuje wartosc do buf (max buflen), zwraca dlugosc w *out_len; -ENOENT brak */
int gh2_btree_lookup(struct gh_dev*, const struct gh2_bptr *root, const struct gh2_key*,
                     void *buf, uint32_t buflen, uint32_t *out_len);
/* insert/update: CoW, zwraca NOWY korzen. val/len. */
int gh2_btree_insert(struct gh_dev*, struct gh2_alloc*, const struct gh2_bptr *root,
                     uint64_t gen, const struct gh2_key*, const void *val, uint32_t len,
                     struct gh2_bptr *out_root);
/* delete: CoW, zwraca NOWY korzen; -ENOENT brak */
int gh2_btree_delete(struct gh_dev*, struct gh2_alloc*, const struct gh2_bptr *root,
                     uint64_t gen, const struct gh2_key*, struct gh2_bptr *out_root);
/* iteracja w porzadku klucza: callback(key,val,len,ctx); przerwij gdy !=0 */
int gh2_btree_iterate(struct gh_dev*, const struct gh2_bptr *root,
                      int (*cb)(const struct gh2_key*, const void*, uint32_t, void*), void *ctx);
```

## CoW (kluczowa semantyka)
Mutacja schodzi do liścia, kopiując ścieżkę: każdy odwiedzony węzeł → nowy blok (alloc),
zmodyfikowany, rodzic dostaje nowy `child` bptr, w górę do korzenia. Stare bloki ścieżki →
`alloc->free` (w v2.2 dekrement refcount; jeśli współdzielone ze snapshotem, NIE zwalniane).
**Stary korzeń pozostaje w pełni ważny** (niezmienione bloki współdzielone). Split (insert)
i merge/borrow (delete) też CoW.

## Algorytmy (skrót; szczegóły w planie, budowa etapami)
- **Insert:** zejdź do liścia (CoW ścieżki); wstaw item posortowany; jeśli liść przepełniony
  → split na dwa, promuj klucz-rozdzielający do rodzica (CoW); jeśli korzeń się dzieli →
  nowy korzeń o 1 poziom wyżej.
- **Delete:** zejdź do liścia (CoW); usuń item; jeśli liść < MIN → borrow od sąsiada lub
  merge (CoW); propaguj w górę; jeśli korzeń wewnętrzny ma 1 dziecko → obniż wysokość.
- **Lookup/iterate:** schodzenie po kluczu / in-order DFS.

## Testy (BRAMKA) — property-based vs model referencyjny
`tests/test_v2btree.c`:
1. **Model referencyjny:** posortowana tablica `{key,val}` w teście. Losowa sekwencja
   insert/update/delete/lookup (np. 5000 operacji, klucze z małej i dużej przestrzeni by
   wymusić kolizje i split/merge). Po KAŻDEJ operacji: lookup wszystkich kluczy modelu ==
   drzewo; iteracja drzewa == model posortowany; brakujące klucze → -ENOENT.
2. **Wymuszone split/merge:** wstaw rosnąco i losowo dużo itemów (wiele poziomów); usuń
   większość (wymusza merge/borrow, obniżenie wysokości); niezmienniki trzymają.
3. **CoW/snapshot foundation:** zapamiętaj root R1; zrób serię insertów → R2; lookup w R1
   zwraca STARY stan, w R2 nowy (stary korzeń nietknięty).
4. **Wartości zmiennej długości:** itemy 1..~kilkaset B; update zmieniający długość.
5. **Sumy/odczyt:** przekręć bit w bloku węzła (przez bptr.block) → odczyt -EIO (dup=0).
6. **Pojemność:** wartość za duża → -EFBIG; pełny liść split działa.
ASan/UBSan czyste; brak wycieków bloków (każdy alloc ma free w CoW; licznik w stubie zgadza
się po sekwencji).

## Świadome decyzje
- Klucze unikalne (kolizje hash itemów katalogu rozwiąże v2.3 przez offset w kluczu).
- Wartości małe (≤ pojemność liścia); duże dane plików = ekstenty (v2.4), nie B-drzewo.
- Refcount/sharing przy free → v2.2; v2.1 stub free od razu zwalnia (brak snapshotów jeszcze).
- Merge/borrow pełne (nie tylko free-on-empty) — jakość przede wszystkim, bez skrótów.
