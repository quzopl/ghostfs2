# ghostfs v2.1 — CoW B-tree: plan implementacji (etapowy)

> REQUIRED: superpowers:subagent-driven-development. Budowa PRZYROSTOWA — B-drzewo musi być
> testowane na każdym etapie. Bramka każdego zadania: property-based vs model referencyjny +
> ASan + brak wycieków bloków. Pliki: `src/v2/gh2_btree.{h,c}`, `tests/test_v2btree.c`.

**Goal:** Generyczne CoW B-drzewo (klucz→wartość zmiennej długości) zgodne ze spec
`docs/superpowers/specs/2026-06-25-ghostfs-v2.1-btree-design.md`.

**Wspólne:** I/O przez `gh_disk_read/write` (szyfrowanie+cache, BEZ txn/csum v1); sumy v2 w
`gh2_bptr` (gh_crc32 plaintextu); węzły memset 0 przed wypełnieniem (determinizm sumy);
alokator pluggable (stub bump+freelist w teście/module). `-Isrc` już w CFLAGS; `src/v2/*.c`
w CORE. Każdy etap: `make build/test_v2btree && ./build/test_v2btree` + wariant ASan.

---

## Task 1: Format węzła + alokator-stub + liść (insert/lookup/iterate, BEZ split)

**Files:** Create `src/v2/gh2_btree.h`, `src/v2/gh2_btree.c`; Test `tests/test_v2btree.c`.

- [ ] Struktury w `gh2_btree.h`: `gh2_key`, `gh2_node_hdr`, `gh2_leaf_item`,
  `gh2_internal_ptr`, `gh2_alloc`, deklaracje API (gh2_key_cmp, gh2_node_read/write,
  gh2_btree_create/lookup/insert/delete/iterate, gh2_bump_alloc helpery). `_Static_assert`
  rozmiarów. Stałe pojemności/MIN.
- [ ] `gh2_btree.c`: `gh2_key_cmp`; `gh2_node_read`/`gh2_node_write` (csum+dup+gh_disk_*);
  stub alokatora `gh2_bump_alloc` (struct z next_free + tablica wolnych + alloc/free + licznik
  „w użyciu" do detekcji wycieków); `gh2_btree_create` (pusty liść); `gh2_btree_lookup`
  (na razie tylko liść-korzeń: szukaj w liściu); `gh2_btree_insert` (TYLKO gdy mieści się w
  liściu-korzeniu: wstaw posortowany item, update gdy klucz istnieje, CoW liścia → nowy root;
  `-EFBIG` gdy się nie mieści — split w Task 2); `gh2_btree_iterate` (liść in-order).
- [ ] Test: insert/lookup/update/iterate na pojedynczym liściu vs model (tablica posort.);
  wartości zmiennej długości; update zmieniający długość; -ENOENT; -EFBIG przy przepełnieniu;
  CoW (stary root widzi stary stan); wyciek bloków = 0 (licznik stuba). + ASan.
- [ ] `make build/test_v2btree && ./build/test_v2btree` zielone; ASan zielone. Commit.

## Task 2: Split + węzły wewnętrzne (drzewo wielopoziomowe: insert/lookup/iterate)

**Files:** Modify `src/v2/gh2_btree.c` (+ `.h` jeśli trzeba); Test `tests/test_v2btree.c`.

- [ ] Insert z **podziałem liścia** przy przepełnieniu: split na dwa ~równe, promuj
  najmniejszy klucz prawego do rodzica (CoW). Węzły **wewnętrzne**: lookup schodzi po kluczu
  (największy `i` z `key[i] <= szukany`); insert do dziecka, przy promocji wstaw do
  wewnętrznego, split wewnętrznego przy przepełnieniu, **nowy korzeń** gdy korzeń się dzieli
  (wysokość +1). `gh2_btree_iterate` — DFS in-order przez wszystkie poziomy.
- [ ] Test: insert DUŻO (rosnąco, malejąco, losowo — wymusza wiele split i poziomów) vs
  model; lookup wszystkich; iteracja == posortowany model; klucze nieobecne → -ENOENT;
  losowy ciąg 5000 insert/update + walidacja po każdej; CoW snapshot (R1 vs R2); wyciek=0; ASan.
- [ ] Zielone + ASan. Commit.

## Task 3: Delete + merge/borrow (pełne rebalansowanie)

**Files:** Modify `src/v2/gh2_btree.c`; Test `tests/test_v2btree.c`.

- [ ] `gh2_btree_delete`: zejdź do liścia (CoW), usuń item; jeśli liść < MIN → **borrow** od
  sąsiada (przesuń item + popraw klucz-rozdzielający w rodzicu) lub **merge** z sąsiadem (CoW),
  usuń wskaźnik z rodzica; propaguj niedobór w górę; jeśli korzeń wewnętrzny zostaje z 1
  dzieckiem → **obniż wysokość** (dziecko nowym korzeniem). Free starych bloków (stub).
- [ ] Test (property-based, BRAMKA): losowy ciąg ~10000 operacji insert/delete/update/lookup
  z mieszanej przestrzeni kluczy; po KAŻDEJ: pełna zgodność z modelem (lookup wszystkich +
  iteracja == model); usuń do pusta (wysokość wraca do 1, liść 0 itemów); naprzemienne
  insert/delete utrzymujące drzewo na granicy split/merge; wyciek bloków = 0 po sekwencji
  (każdy CoW free zwalnia stare); ASan/UBSan czyste; przekręcony bit węzła → -EIO.
- [ ] Zielone + ASan + property 10000 ops zielone. Commit.

## Self-Review (plan)
**Pokrycie spec:** format+I/O+alokator (T1) ✓; insert/lookup/iterate+split+internal (T1/T2) ✓;
delete+merge/borrow (T3) ✓; CoW/snapshot foundation (testy T1/T2) ✓; sumy/dup-read (T1) ✓;
property-based vs model (T1/T2/T3) ✓; wycieki bloków (stub licznik) ✓.
**Ryzyka:** (1) B-drzewa są bardzo podatne na błędy → property-based vs model referencyjny po
KAŻDEJ operacji jest bramką (łapie split/merge/borrow/CoW bugi); (2) determinizm sumy →
memset 0 węzła; (3) wycieki bloków przy CoW → licznik „w użyciu" w stubie = 0 po sekwencji;
(4) CoW poprawność (stary root ważny) → test snapshot R1/R2. Build przyrostowy: każdy etap
zielony zanim następny.
